Browser-basierter Zugriff auf isolierte Malware-Analyse-Infrastruktur ohne Client-Installation. Entwickelt fuer regulierte Umgebungen mit vollstaendiger Audit-Faehigkeit.

Executive Summary

Fuer CISOs und Security-Entscheider

🛡

Das Problem

Malware-Analyse erfordert isolierte Umgebungen. Der Zugriff auf diese Systeme ist oft komplex, erfordert VPN-Clients und birgt Risiken bei der Credential-Verwaltung.

💡

Die Loesung

IceLaborVPN bietet sicheren, browser-basierten Zugriff mit Multi-Faktor-Authentifizierung. Keine Software-Installation auf Client-Seite erforderlich.

✅

Der Nutzen

Reduzierte Angriffsflaeche, vollstaendige Audit-Trails, DORA-Konformitaet und einfache Verwaltung fuer Security-Teams weltweit.

Kernfunktionen

Entwickelt fuer Enterprise-Sicherheitsanforderungen

🔒

Zero-Trust Architektur

WireGuard VPN-Mesh mit Headscale Control Plane. Jede Verbindung wird authentifiziert und autorisiert - kein implizites Vertrauen.

  • Ende-zu-Ende-Verschluesselung
  • Mikrosegmentierung
  • Identity-basierter Zugriff
🖥

HTML5 Remote Access

SSH, VNC und RDP direkt im Browser ueber Apache Guacamole. Keine Client-Software, keine Firewall-Ausnahmen fuer Endgeraete.

  • Kein Client-Download
  • Plattformunabhaengig
  • Copy/Paste kontrolliert
🔐

Multi-Faktor-Authentifizierung

TOTP/2FA ist obligatorisch fuer alle Benutzer. Integration mit gaengigen Authenticator-Apps (Google, Microsoft, Authy).

  • TOTP (RFC 6238)
  • Backup-Codes
  • Account-Lockout
📹

Session Recording

Vollstaendige Aufzeichnung aller Remote-Sessions fuer Compliance und Forensik. Wiederabspielbar fuer Audits.

  • Video-Aufzeichnung
  • Keystroke-Logging
  • 1 Jahr Retention
🚫

Brute-Force-Schutz

Mehrstufige Verteidigung gegen Credential-Angriffe: Rate Limiting, Guacamole-Lockout und Fail2ban IP-Blocking.

  • 5 Logins/Minute
  • Automatische IP-Sperre
  • Echtzeit-Alerting
🔔

Echtzeit-Benachrichtigungen

Pushover-Integration fuer sofortige Security-Alerts: Erfolgreiche Logins, Bans, erkannte Angriffe und Service-Ausfaelle.

  • Login-Benachrichtigungen
  • Angriffs-Warnungen
  • Service-Monitoring

Systemarchitektur

Defense-in-Depth Design

Komponenten-Uebersicht

Nginx Reverse Proxy mit TLS 1.3, Rate Limiting und Security Headers
Guacamole HTML5 Gateway fuer SSH, VNC, RDP mit TOTP und Session Recording
Headscale Self-hosted Tailscale Control Server fuer WireGuard VPN
Fail2ban Intrusion Prevention mit automatischer IP-Blockierung
PostgreSQL Datenbank fuer Benutzer, Sessions und Audit-Logs

Sicherheitsarchitektur

Defense in Depth

1

Netzwerksicherheit

  • Firewall (UFW) - Nur Ports 80, 443 offen
  • Rate Limiting - 5 Logins/Min, 30 Req/Sek
  • Fail2ban - Automatische IP-Sperre nach 5 Fehlversuchen
2

Transportverschluesselung

  • TLS 1.3 - Modernste Verschluesselung
  • HSTS - Erzwungenes HTTPS
  • WireGuard - Curve25519, ChaCha20-Poly1305
3

Authentifizierung

  • Passwort - Min. 12 Zeichen, SHA-256
  • TOTP/2FA - Obligatorisch fuer alle Benutzer
  • Session Timeout - 60 Minuten Inaktivitaet
4

Autorisierung

  • RBAC - Rollenbasierte Zugriffskontrolle
  • Connection-Level - Granulare Berechtigungen
  • Zeitbasiert - Optionale Zugriffszeiten
5

Audit und Monitoring

  • Access Logging - Alle Zugriffe protokolliert
  • Session Recording - Video-Aufzeichnung
  • Echtzeit-Alerting - Pushover-Benachrichtigungen

Installation

Schnellstart-Anleitung

1

Voraussetzungen

  • Ubuntu 22.04 LTS oder Debian 12
  • Oeffentliche IPv4-Adresse
  • DNS A-Record fuer Ihre Domain
  • Min. 2 CPU Cores, 4 GB RAM, 50 GB SSD
2

Repository klonen

git clone https://github.com/icepaule/IceLaborVPN.git
cd IceLaborVPN
3

Konfiguration anpassen

cp .env.example .env
nano .env  # Alle Werte ausfuellen!

Wichtige Variablen: HEADSCALE_DOMAIN, GUAC_ADMIN_PASSWORD, PUSHOVER_APP_TOKEN

4

Installation starten

chmod +x scripts/install.sh
sudo ./scripts/install.sh
5

Erster Login

  1. Browser oeffnen: https://ihre-domain.com/guacamole/
  2. Mit Admin-Credentials einloggen
  3. QR-Code mit Authenticator-App scannen
  4. 6-stelligen TOTP-Code eingeben

Post-Installation Checkliste

  • SSL-Zertifikat validieren
  • TOTP fuer Admin-Account einrichten
  • Backup-Job konfigurieren
  • Monitoring-Alerts einrichten
  • Fail2ban-Regeln testen
  • Penetration Test durchfuehren

# Fail2ban Status
sudo fail2ban-client status guacamole

# IP entsperren
sudo fail2ban-client set guacamole unbanip 192.0.2.1

# Alle Sessions beenden (Notfall)
sudo docker exec guacamole-db psql -U guacamole -d guacamole \
  -c "DELETE FROM guacamole_user_session;"

Haeufige Fragen

FAQ fuer Administratoren

Stellen Sie sicher, dass die Systemzeit synchronisiert ist: timedatectl. TOTP ist zeitbasiert und erfordert eine Abweichung von maximal 30 Sekunden. Bei anhaltendem Problem kann der TOTP-Code in den Benutzereinstellungen zurueckgesetzt werden.

sudo fail2ban-client set guacamole unbanip 192.0.2.1

Ersetzen Sie die IP-Adresse entsprechend. Pruefen Sie vorher, ob die IP legitimerweise geblockt wurde.

1. AuthKey generieren: /opt/IceLaborVPN/scripts/headscale-onboard.sh --generate-key

2. Auf dem neuen System: sudo tailscale up --login-server https://ihre-domain.com --authkey <key>

3. Connection in Guacamole anlegen

Session Recordings werden unter /opt/guacamole/record/ gespeichert. Sie koennen mit dem Guacamole Player wiedergegeben werden. Retention: 1 Jahr.
